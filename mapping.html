Conceptual Mapping to INSPIRE Data Specifications

After the initial analysis, we created conceptual mappings from each of the six selected source data sets to the appropriate INSPIRE GML Application schemas, taking into account the rules set out by the data specifications.

As there were no previous mappings like Matching Tables available, we started the mapping from scratch and directly created executable hale studio mappings. The process to create such a mapping is as follows:

  1. Map Types: Select source and target types (e.g. tables, classes, documents) and decide which relation they have:
    • Retype: Create one target object for each source object.
    • Split: Create one or more target objects for each source object.
    • Merge: Create one target object from one or more source objects of the same type.
    • Join: Create one target object from many source objects of different types.
